    go语言常见坑

    1.由变量初始化顺序导致的nil错误

    全局变量的加载顺序优先于局部变量,在全局变量处使用Getter方法获取另一个包中的全局变量(使用变量逃逸的方式进行赋值),可能会由于加载顺序问题,导致全局变量还未被赋值,就已被引用,导致空指针异常.

    2.使用go build或go run main.go routers.go得到的二进制可执行文件的文件名不一致

    使用前者,生成的二进制可执行文件与项目目录名一致,使用后一种,得到的二进制可执行文件名为main.
